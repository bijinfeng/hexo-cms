# Hexo CMS — 路线图

> **最后更新**: 2026-05-16
> **当前版本**: 0.1.0 (MVP)

---

## 概述

MVP 已覆盖博客内容管理的核心链路：登录 → 导入仓库 → 文章/页面/媒体 CRUD → 部署 → 主题切换。插件系统和桌面端基础设施已就绪。接下来分两个方向推进：

1. **MVP 收尾** — 补齐明显缺口，达到可对外发布的质量标准
2. **后续迭代** — 按优先级实现增强功能和生态建设

---

## 一、MVP 收尾 (v0.1.x)

这些是 MVP 阶段遗留的缺口，应该在正式发布前补齐。

### 🔴 高优先级

| 功能 | 工作量 | 说明 |
|---|---|---|
| **评论系统集成** | 3-5 天 | 目前使用 mock 数据。推荐先接入 Giscus（基于 GitHub Discussions，零成本，无需额外服务）。备选 Waline。 |
| **菜单/导航管理** | 3-4 天 | PROJECT_PLAN 第 10 项。可视化编辑导航菜单，拖拽排序，多级菜单。需要新增 `/menus` 页面和对应的 API/IPC。 |
| **E2E 测试** | 5-7 天 | Playwright 覆盖核心流程：登录 → 导入仓库 → 创建文章 → 编辑 → 删除 → 部署。保障关键路径不被回归破坏。 |

### 🟡 中优先级

| 功能 | 工作量 | 说明 |
|---|---|---|
| **自动保存草稿** | 1-2 天 | 编辑器中定时将内容存到 localStorage，防止意外关闭丢失内容。页面刷新后恢复。 |
| **字数统计 + 阅读时间** | 0.5-1 天 | 编辑器底部显示字数和预估阅读时间，提升编辑体验。 |
| **macOS 代码签名** | 2-3 天 | 桌面端目前未签名，用户打开 DMG 需手动允许。配置 Apple Developer 证书签名和公证。 |
| **编辑器偏好设置** | 1-2 天 | Settings 中添加编辑器设置：字体大小、主题（亮/暗跟随系统）、自动保存间隔。 |

### 🟢 低优先级

| 功能 | 工作量 | 说明 |
|---|---|---|
| **标签合并功能** | 1 天 | Tags 页中添加 "合并到..." 操作，将一个标签的所有文章批量迁移到另一个标签。 |
| **媒体批量删除** | 0.5-1 天 | MediaPage 添加多选和批量删除，与 PostsPage 批量操作一致。 |

## 二、短期迭代 (v0.2 — v0.3)

MVP 收尾后的下一批功能，按价值和复杂度排序。

### 编辑器增强

| 功能 | 工作量 | 价值 | 说明 |
|---|---|---|---|
| **文章版本历史** | 3-4 天 | 高 | 基于 git commit 历史展示文章修改记录，支持 diff 对比和回退。增强用户信心，不怕误改。 |
| **模板/草稿模板** | 1-2 天 | 中 | 预设 frontmatter 模板（技术文章/随笔/周报），新建时一键选择。 |
| **全屏专注模式** | 0.5-1 天 | 中 | 隐藏侧边栏和顶栏，编辑器全屏，提升沉浸式写作体验。 |

### 部署增强

| 功能 | 工作量 | 价值 | 说明 |
|---|---|---|---|
| **部署状态实时推送** | 2-3 天 | 高 | 通过 SSE 或轮询实时展示部署进度，替代目前的手动刷新。 |
| **部署日志查看** | 2 天 | 中 | 在 DeployPage 中直接查看 GitHub Actions 运行日志（需 API 权限扩展）。 |
| **部署回滚** | 3-4 天 | 中 | 基于 git revert 实现一键回滚到指定版本。 |

### 内容管理增强

| 功能 | 工作量 | 价值 | 说明 |
|---|---|---|---|
| **分类树形结构** | 1-2 天 | 中 | Tags 页面中分类 Tab 改为树形展示（支持父子分类），更直观。 |
| **提交历史查看** | 2-3 天 | 中 | 新增 `/commits` 页面，展示仓库最近提交记录。 |
| **回收站** | 2-3 天 | 中 | 删除文章先移到 `_drafts/` 或标记 `deleted: true`，而非直接删除，降低误操作风险。 |

### 插件生态

| 功能 | 工作量 | 价值 | 说明 |
|---|---|---|---|
| **SEO Inspector 插件** | 3-4 天 | 高 | 文章编辑器中实时检查 SEO 问题：标题长度、描述缺失、关键词密度、Open Graph 标签等。 |
| **Draft Coach 插件** | 2-3 天 | 中 | 草稿质量检查：段落长度、阅读难度、链接有效性、错别字提示。 |
| **Link Checker 插件** | 2-3 天 | 中 | 扫描全站文章中的外部链接，检测失效链接。 |

### 主题系统

| 功能 | 工作量 | 价值 | 说明 |
|---|---|---|---|
| **主题配置可视化** | 3-4 天 | 中 | 解析 `_config.theme.yml`，生成表单 UI，可视化编辑主题参数。 |
| **在线主题安装** | 2-3 天 | 低 | 从 npm/themes 仓库安装新主题（git clone 到 themes/ 目录）。 |

## 三、中期规划 (v0.4 — v0.5)

需要较大投入但能显著提升产品竞争力的功能。

### 数据分析

| 功能 | 工作量 | 价值 | 说明 |
|---|---|---|---|
| **Analytics Dashboard 插件** | 5-7 天 | 高 | 接入 Umami 或 Plausible API，在仪表板展示 PV/UV/来源/热门文章图表。依赖插件 network.fetch 代理。 |

### 多分支工作流

| 功能 | 工作量 | 价值 | 说明 |
|---|---|---|---|
| **分支管理** | 5-7 天 | 中 | 支持在多个分支间切换编辑。典型场景：`draft` 分支编辑，合并到 `master` 触发部署。 |

### 移动端适配

| 功能 | 工作量 | 价值 | 说明 |
|---|---|---|---|
| **响应式布局** | 5-7 天 | 中 | 移动端适配所有页面，支持手机端快速发布/编辑。响应式侧边栏改为底部 Tab 导航。 |

### 协作功能

| 功能 | 工作量 | 价值 | 说明 |
|---|---|---|---|
| **多用户支持** | 5-7 天 | 高 | 支持多人通过各自 GitHub 账号登录同一站点，权限隔离（管理员/编辑/作者）。 |
| **评论回复** | 3-4 天 | 中 | 在 CMS 内直接回复评论（依赖评论系统 API）。 |

## 四、远期愿景 (v0.6+)

战略级功能，需要较长的技术预研和实现周期。

### 插件生态开放

| 功能 | 说明 |
|---|---|
| **第三方插件沙箱** | iframe/Web Worker/独立进程隔离，确保第三方插件不影响核心稳定性。 |
| **插件市场/注册中心** | 在线浏览、安装、评分插件。需要后端服务（注册中心 API + 存储）。 |
| **插件签名与审核** | 插件签名验证 + 人工审核流程，保障安全。 |

### 企业功能

| 功能 | 说明 |
|---|---|
| **SSO/企业认证** | 支持 OIDC/SAML，适配企业 GitHub Enterprise。 |
| **RBAC 权限系统** | 细粒度角色权限（超级管理员/编辑/作者/只读）。 |
| **审计日志** | 记录所有操作历史，支持回溯和合规。 |
| **多站点管理** | 一个 CMS 实例管理多个 Hexo 博客仓库。 |

### 技术演进

| 方向 | 说明 |
|---|---|
| **数据库迁移到 Turso** | 从本地 SQLite 迁移到 Turso（边缘数据库），支持多区域部署。 |
| **PWA 支持** | Web 端添加 PWA，支持离线编辑、本地缓存、推送通知。 |
| **AI 写作辅助** | 集成 LLM，提供续写、摘要、翻译、润色等功能。 |

## 技术债务跟踪

以下技术债务应在迭代中逐步消化：

| 债务 | 优先级 | 说明 |
|---|---|---|
| `any` 类型消除 | 中 | 代码中 ~15 处 `any` 类型，应逐步替换为具体类型 |
| API 输入验证（Zod） | 中 | 目前 API 路由缺少运行时参数验证 |
| 性能优化 | 低 | 大文件列表虚拟化、图片懒加载、bundle size 优化 |
| 国际化（i18n） | 低 | 目前仅中文，后续可考虑中英双语 |

# Topbar 功能设计：搜索 / 通知 / 用户头像

> 日期：2026-05-15
> 状态：已确认

## 背景

Dashboard 页面 Topbar 中的搜索框、通知铃铛、用户头像目前都是静态占位符，无实际功能。本次设计评审这三个功能的去留和交互形态。

## 产品形态

Hexo CMS 是**单用户**博客管理工具，用户通过 GitHub OAuth 登录，所有数据操作通过 GitHub API 完成。没有多用户协作场景。

---

## 功能一：搜索 → 命令面板 (Command Palette)

### 决策：保留，升级为命令面板

### 交互形态

参照 VS Code / Linear / GitHub 的 ⌘K 命令面板模式：

- **触发**：点击搜索框 或 `⌘K`（Mac）/ `Ctrl+K`（Windows）
- **呈现**：Modal overlay 居中弹出，半透明遮罩背景
- **内容分组**：

| 分组 | 内容 |
|------|------|
| 导航 | 文章列表 / 页面管理 / 媒体库 / 标签管理 / 主题设置 / 部署管理 / 系统设置 |
| 操作 | 新建文章 / 新建页面 / 触发部署 |
| 最近文章 | 最近编辑的 5 篇文章（标题 + 日期） |

- **交互**：
  - 默认列出所有命令 + 最近文章
  - 输入即模糊搜索，同时匹配命令名称和文章标题
  - ↑↓ 键导航，Enter 执行，Esc 关闭
  - 点击遮罩关闭
  - 分组标题不可选中，仅作视觉分隔
- **键盘快捷键提示**：有快捷键的命令在右侧显示快捷键标签（如 `⌘1`）

---

## 功能二：通知铃铛

### 决策：保留代码，功能暂不启用

### 说明

当前产品是单用户 CMS，没有评论、没有协作事件，不存在需要实时通知的场景。部署结果在 Deploy 页面已有反馈。

**代码保留**：Topbar 中通知铃铛的 DOM 和样式代码不动。**未来接入评论插件**（如 Giscus / Waline webhook）时，通知中心可重新激活，用于展示：
- 新评论通知
- 垃圾评论拦截通知
- 部署状态通知

届时需将当前静态红点替换为实时未读数，并添加下拉通知列表。

---

## 功能三：用户头像下拉菜单

### 决策：保留，补完标准交互

### 交互形态

参照 GitHub / Vercel 用户菜单模式：

- **触发**：点击头像
- **头像来源**：GitHub OAuth session 中的 `user.avatarUrl`；无头像时 fallback 为首字母圆圈
- **菜单内容**：

| 菜单项 | 行为 |
|--------|------|
| 头像 + 用户名 + 邮箱 | 只读展示 |
| 分隔线 | — |
| 系统设置 | 导航到 `/settings` |
| 退出登录 | 清除 session，跳转登录页 |

- **交互**：
  - 菜单 240px 宽，右对齐于头像
  - 点击菜单项执行对应操作并关闭菜单
  - 点击菜单外部关闭
  - 退出登录无二次确认（单用户 CMS 工具，简化操作流）
  - 桌面端退出登录走现有 IPC 通道清理 token

## 实施范围

| 功能 | 工作内容 | 涉及文件 |
|------|---------|---------|
| 命令面板 | 新建 CommandPalette 组件，集成 `⌘K` 快捷键，接入路由和数据 | `packages/ui/src/components/command-palette.tsx`、`Topbar.tsx` |
| 通知 | 无代码改动，在 Topbar.tsx 添加注释说明预留用途 | `Topbar.tsx` |
| 用户菜单 | 接入真实 GitHub 头像，实现下拉菜单（设置、退出登录） | `Topbar.tsx` |

## 不做的事

- 通知实时推送 / WebSocket 连接
- 命令面板中的全文文章内容搜索（posts 页面已有独立搜索）
- 用户菜单中的个人资料编辑（settings 页面已有）
- 退出登录二次确认弹窗
